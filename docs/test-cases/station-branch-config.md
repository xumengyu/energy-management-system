## 适用范围

本用例覆盖 **支路配置页** 的入口导航、支路新建/编辑/删除、计量电表绑定、未配置设备与支路间的拖拽划分、保存与再次进入、中英文界面及边界行为。

不包含：一次接线图 [`StationArchitecture.tsx`](../../components/StationArchitecture.tsx) 中与「支路表」相关的纯展示布局（该处为图示语义，非本配置页）。

## 产品文案要点（中文，验收对照）

与界面展示一致的主要文案键（节选）：

- 页面标题：`支路配置`；副标题见 `subtitle`（说明弹框编辑、删除后设备回未配置、可拖拽）。
- 左侧：`支路列表`、`新建支路`、`暂无支路，请先新建支路后再分配设备。`
- 右侧：`未配置设备`；空池：`当前没有未配置设备，均已划入支路。`；拖拽提示：`拖回此处以从支路撤出`
- 支路卡片：`支路名称`、`绑定计量电表`、`未命名支路`、`已挂接设备`、`将设备拖放到此处以挂接到该支路`
- 弹窗：`新建支路` / `编辑支路`、`删除支路` 及确认文案中的 `{name}` 占位

## 已知限制（测试环境说明）

1. **持久化**：支路配置保存在应用内存 `feederConfigsByStation` 中（[`App.tsx`](../../App.tsx)）。**整页刷新后配置丢失**，与是否点击「保存配置」无关（当前无 localStorage/API 持久化）。
2. **路由形态**：当前应用以 React 状态驱动 `currentPath`，浏览器地址栏未必与 `/stations/branches` 同步。以下「支路页」均指应用内已导航到该路径且已选中站点上下文的情形。
3. **无有效站点上下文**：当 `branchConfigStationId` 为空或找不到对应站点时，[`App.tsx`](../../App.tsx) 在 `/stations/branches` 分支下会 **渲染电站列表** 而非支路配置页。
4. **设备列表**：由 `buildStationDevices` 生成——含站点 `deviceTypes` 对应设备；无类型时含「站点汇总设备」；并固定包含多条 `demo::` 演示设备（见 [`StationBranchConfig.tsx`](../../components/StationBranchConfig.tsx)）。
5. **电表选项**：固定 6 项（`MTR-PCC`、`MTR-FEED-A`、`MTR-FEED-B`、`MTR-LOAD`、`MTR-PV`、`MTR-ESS`），中英文标签随语言切换。

## 前置条件（通用）

- 应用已启动，可进入「电站」列表页（`/stations`）。
- 默认使用中文界面；英文用例单独注明切换语言（顶栏语言菜单）。
- 测试 **保存后再进入** 类用例时，**勿刷新页面**，否则内存配置丢失，无法验证加载。

## 实现对照摘要（供自动化/回归引用）

| 行为 | 代码位置 |
|------|----------|
| 保存时上送 `{ branches, assignments }` | `StationBranchConfig` → `handleSave` → `onSave` |
| 内存合并 `feederConfigsByStation` | `App.tsx` → `onSave={(stationId, config) => setFeederConfigsByStation(...)}` |
| 删除支路清空挂接 | `removeBranch` 内遍历 `assignments`，值等于该 `branchId` 的键置为 `''` |
| 新建支路 ID | `br-${Date.now()}` |
| 拖拽 MIME | `text/plain`（常量 `DND_DEVICE_MIME`） |
