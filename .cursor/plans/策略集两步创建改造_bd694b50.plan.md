# 策略集两步创建改造方案

## 需求澄清（最新）

点击策略列表中的 **【编辑】** 时，弹框内应显示 **「创建策略」页面的那些内容**，即与 `[components/StrategyEditor.tsx](file:///Users/mengyu/code/energy-management-system/components/StrategyEditor.tsx)` 一致的表单与交互（名称、执行年/月/日、BESS、功率曲线、启用开关、底部保存/取消等），而不是当前 `[StrategyManager](file:///Users/mengyu/code/energy-management-system/components/StrategyManager.tsx)` 里仅含「策略名称 + 状态」的简易弹框，也**不是** `[CreateStation](file:///Users/mengyu/code/energy-management-system/components/CreateStation.tsx)` 创建设电站表单。

## 实现思路

1. **复用 StrategyEditor**
  - 在 `StrategyManager` 的「编辑策略」弹框中，用可滚动容器（如 `max-h-[85vh] overflow-y-auto`）包裹 `StrategyEditor`。
  - 传入：`lang`、`theme`（`StrategyManager` 已有）、`onBack` 关闭弹框、`onSave` 关闭弹框并可选回写 mock 数据。
2. **Props 扩展（建议）**
  - 为 `StrategyEditor` 增加可选 `initialData?: Partial<...>` 或 `strategyId?: string`，用于在编辑时预填名称、年、月、功率曲线等（当前若无可先 mock，与列表行对齐）。
  - 增加可选 `embedded?: boolean`：为 `true` 时去掉外层 `max-w-7xl` 全页感、或压缩顶部标题区，避免弹框内双重「页面标题」冲突。
3. **移除或替换**
  - 删除或不再使用 `showEditStrategyModal` 内现有的「编辑策略详情」表单（`editingStrategyName` / `editingStrategyStatus` 等），避免两套 UI 并存。
4. **路由与头部**
  - 弹框内编辑不跳转 `/strategy/create`；若用户仍从侧栏进入创建策略页，行为保持现有独立页。

## 涉及文件

- `[components/StrategyManager.tsx](file:///Users/mengyu/code/energy-management-system/components/StrategyManager.tsx)`：弹框内容替换为 `StrategyEditor`；编辑按钮 `onClick` 打开弹框并 `setEditingStrategyRef`。
- `[components/StrategyEditor.tsx](file:///Users/mengyu/code/energy-management-system/components/StrategyEditor.tsx)`：可选 `embedded` + `initialData`；`useEffect` 根据 `initialData` 填充 state。

## 验收

- 在策略列表点击 **编辑**：弹框内出现与「创建策略」页一致的完整表单。
- 关闭/取消：弹框关闭，不强制路由跳转。
- 保存：弹框关闭（若后续有持久化再接入）。
